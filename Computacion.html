Introducción

En el marco del proyecto aula "Escuela Verde con Camino a la Sustentabilidad", la asignatura de Computación Básica II tuvo un rol fundamental como plataforma integradora de todos los contenidos generados por las distintas materias participantes. Nuestra misión fue desarrollar una herramienta tecnológica que permitiera documentar, organizar y compartir de manera accesible y atractiva los diversos aportes al proyecto.

La página web elaborada representa no solo un producto final, sino también un proceso de aprendizaje donde aplicamos conocimientos fundamentales de desarrollo web utilizando HTML5 y CSS básico. Este proyecto nos permitió comprender cómo la tecnología puede servir como puente entre diferentes disciplinas académicas y como herramienta para la difusión de iniciativas sustentables.

Objetivos del Proyecto

Los objetivos específicos de nuestra participación fueron:

  • Diseñar y desarrollar una página web responsive que integrara todos los contenidos del proyecto aula
  • Aplicar los conceptos de estructura semántica HTML aprendidos en clase
  • Implementar estilos CSS básicos para mejorar la presentación visual
  • Organizar la información de manera lógica y accesible
  • Crear un sistema de navegación interna entre secciones
  • Optimizar imágenes y recursos multimedia para web
  • Trabajar colaborativamente usando metodologías básicas de desarrollo web

Proceso de Desarrollo

El desarrollo del sitio web siguió una metodología por etapas:

1. Análisis de Requerimientos

  • Reunión con representantes de otras materias para entender sus necesidades
  • Definición de la estructura general del sitio
  • Identificación de los tipos de contenido a incluir

2. Diseño de la Estructura

  • Creación de un diagrama de flujo del sitio
  • Definición de la jerarquía de información
  • Diseño del sistema de navegación
  • Selección de paleta de colores acorde al tema ecológico

3. Implementación Técnica

  • Maquetación HTML5 semántica con etiquetas como header, section, article, footer
  • Uso de divs para organización del layout
  • Implementación de estilos CSS para:
    • Tipografía (Google Fonts)
    • Colores y fondos
    • Diseño de cajas (márgenes, padding, bordes)
    • Disposición flexible (Flexbox)
  • Inserción y optimización de imágenes
  • Creación de reglas CSS para diferentes tamaños de pantalla

4. Integración de Contenidos

  • Recopilación y organización de textos proporcionados por otras materias
  • Formateo consistente de encabezados y párrafos
  • Creación de listas ordenadas y no ordenadas
  • Implementación de enlaces internos y externos

5. Pruebas y Ajustes

  • Validación del código HTML
  • Pruebas de visualización en diferentes navegadores
  • Ajustes de responsividad
  • Corrección de errores reportados por usuarios de prueba

Tecnologías Utilizadas

Para el desarrollo de este proyecto web utilizamos las siguientes tecnologías:

  • HTML5: Para la estructura y contenido del sitio
    • Etiquetas semánticas (header, section, article, footer)
    • Elementos de formato (h1-h6, p, ul/ol, li)
    • Elementos multimedia (img)
    • Elementos de agrupación (div)
  • CSS: Para el diseño y presentación
    • Selectores de clase
    • Propiedades de texto (font-family, text-align)
    • Modelo de caja (margin, padding, border)
    • Flexbox para diseño flexible
    • Colores hexadecimales y por nombre
  • Google Fonts: Para la tipografía DM Sans
  • Sistemas de Control de Versiones: Uso básico de GitHub para compartir código

Desafíos Técnicos y Soluciones

Durante el desarrollo enfrentamos varios desafíos técnicos:

1. Organización del Layout

Problema: Crear una estructura de dos columnas donde la barra lateral permaneciera fija mientras el contenido principal fuera desplazable.

Solución: Implementamos un contenedor flex con dirección row-reverse y asignamos un ancho fijo a la barra lateral mientras que el contenido principal ocupaba el espacio restante con flex: 1.

2. Responsividad

Problema: Hacer que el sitio se visualizara correctamente en diferentes tamaños de pantalla.

Solución: Usamos unidades relativas como porcentajes para anchos y em/rem para tamaños de fuente, además de media queries básicas para ajustar el diseño en pantallas pequeñas.

3. Integración de Contenido Diverso

Problema: Cada materia proporcionó información en formatos diferentes que necesitaban ser estandarizados.

Solución: Creamos plantillas HTML consistentes para cada sección y normalizamos los estilos CSS para mantener coherencia visual.
